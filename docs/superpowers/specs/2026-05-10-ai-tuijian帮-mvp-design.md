## 1. 产品概述

### 1.1 产品定位

- **产品名**：AI推荐帮
- **产品类型**：Web SaaS 应用
- **核心价值主张**：帮助中小微企业的品牌被AI看见、被AI理解、被AI推荐

### 1.2 目标客户

- 中国境内中小微企业
- 有产品推广营销需求
- 不懂GEO但是想让自己的品牌被AI推荐

### 1.3 成功指标

- 第一年目标：1000付费用户
- MVP验证：3-5个种子用户完成付费闭环

## 2. 商业模式

### 2.1 定价策略

| 套餐 | 价格 | 功能 |
|------|------|------|
| 免费版 | ¥0 | 1次完整检测+基础报告 |

## 3. 功能架构

### 3.1 核心功能闭环（按优先级）

```
用户 → 资料填写 → Query生成 → 检测执行 → 报告输出 → 优化建议 → 订阅转化 → 持续监测
```

#### P0 - 必须有（MVP核心）

1. **落地页**
   - 核心价值展示
   - 引导注册/登录

2. **资料填写**
   - 品牌名（必填）
   - 行业（必填，下拉选择）
   - 产品/服务（必填）
   - 目标城市（必填）
   - 主要竞品（选填，最多3个）
   - 价格带（选填）
   - 核心卖点（选填）

3. **Query生成（四层机制）**
   - 20个通用品牌问题（系统内置）
   - 30个行业模板问题（按行业预置）
   - 30个企业定制问题（基于客户资料生成）
   - 20个竞品/地域/长尾问题
   - 总计100个监测问题

4. **检测执行**
   - 目标平台选择（豆包/千问/DeepSeek/秘塔）
   - 手动检测入口（结果录入）
   - 状态追踪

5. **报告输出**
   - 可见性分数（百分制）
   - 推荐次数 vs 竞品对比
   - 问题清单
   - AI回答原文引用

6. **优化建议**
   - 事实资产清单（缺什么）
   - 整改优先级
   - 内容框架建议

#### P1 - 应该有（完整闭环）

7. **订阅管理**
   - 支付接入
   - 订阅状态
   - 续费提醒

8. **持续监测**
   - 周期性复检（周/月）
   - 趋势对比

#### P2 - 可以有（后续迭代）

9. **竞品对比**
10. **行业榜单**
11. **数据趋势**

### 3.2 业务流程图

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  落地页  │ → │ 资料填写 │ → │Query生成 │ → │ 检测执行 │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
                                                        ↓
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  持续监测 │ ← │ 订阅转化  │ ← │ 优化建议 │ ← │ 报告输出 │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
```

## 4. 数据模型

### 4.1 用户表（users）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | uuid | 主键 |
| email | string | 邮箱（登录用） |

| created_at | timestamp | 创建时间 |
| subscription | string | 订阅状态（free/pro/year） |

### 4.2 品牌表（brands）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | uuid | 主键 |
| user_id | uuid | 外键 |
| name | string | 品牌名 |
| industry | string | 行业 |
| product | string | 产品/服务 |
| city | string | 目标城市 |
| competitors | json | 竞品列表 |
| price_range | string | 价格带 |
| highlights | string | 核心卖点 |
| created_at | timestamp | 创建时间 |

### 4.3 Query表（queries）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | uuid | 主键 |
| brand_id | uuid | 外键 |
| query_text | string | 问题文本 |
| query_type | string | 问题类型（通用/行业/定制/长尾） |
| search_intent | string | 搜索意图 |
| created_at | timestamp | 创建时间 |

### 4.4 检测结果表（results）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | uuid | 主键 |
| query_id | uuid | 外键 |
| brand_id | uuid | 外键 |
| platform | string | AI平台 |
| is_recommended | boolean | 是否被推荐 |
| sentiment | string | 情感（positive/neutral/negative） |
| ai_content | text | AI回答原文 |
| checked_at | timestamp | 检测时间 |

### 4.5 报告表（reports）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | uuid | 主键 |
| brand_id | uuid | 外键 |
| visible_score | integer | 可见性分数 |
| recommended_count | integer | 被推荐次数 |
| competitor_count | integer | 竞品被推荐次数 |
| issues | json | 问题清单 |
| suggestions | json | 优化建议 |
| created_at | timestamp | 创建时间 |

## 5. 页面设计

### 5.1 页面结构

| 路由 | 名称 | 说明 |
|------|------|------|
| `/` | 首页/落地页 | 核心价值展示，引导注册 |
| `/login` | 登录页 | 登录/注册 |
| `/dashboard` | 用户仪表盘 | 品牌列表+状态概览 |
| `/brand/new` | 新建品牌 | 资料填写 |
| `/brand/:id` | 品牌详情 | 查看/编辑品牌信息 |
| `/brand/:id/queries` | Query列表 | 查看生成的100个问题 |
| `/brand/:id/detect` | 检测页面 | 执行检测 |
| `/brand/:id/report` | 报告页面 | 查看检测报告 |
| `/pricing` | 定价页 | 套餐展示 |
| `/profile` | 个人中心 | 账户设置 |

### 5.2 落地页内容

**主要区块**：
1. Hero区：核心价值主张 + CTA按钮
2. 价值说明：解决什么痛点
3. 使用流程：3步说明
4. 客户见证：案例展示（后续填充）
5. 定价信息：套餐对比
6. FAQ
7. Footer

## 7. 技术实现

### 7.1 技术栈

- **框架**：Next.js 14 (App Router)
- **语言**：TypeScript
- **样式**：Tailwind CSS
- **数据库**：PostgreSQL (Supabase)
- **部署**：Vercel
- **AI辅助**：Cursor/Windsurf (Vibe Coding)

### 7.2 第三方服务

- **数据库**：Supabase / Neon
- **支付**：Stripe / 微信支付 / 支付宝
- **邮件**：Resend / SendGrid
- **AI**：OpenAI / 阿里云百炼

### 7.3 开发计划

**Phase 1：MVP（1-2周）**
1. 项目初始化
2. 落地页开发
3. 用户系统（注册/登录）
4. 品牌资料填写
5. Query生成（简化版）
6. 手动检测结果录入
7. 报告输出

**Phase 2：完善（1-2周）**
1. 支付订阅
2. 持续监测
3. 竞品对比

**Phase 3：迭代（持续）**
1. 行业模板沉淀
2. 自动化检测
3. 行业榜单
